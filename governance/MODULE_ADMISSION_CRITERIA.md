# MODULE_ADMISSION_CRITERIA.md

## Rule

A new module may be added only if it clearly supports at least one of:

1. scientific correctness
2. user learning
3. reproducibility
4. evidence quality
5. TUBITAK proposal readiness
6. auditability
7. execution clarity
8. Codex discipline

## Anti-Fake-Progress Rule

More folders, templates, automation, dashboards, or generated prose are not progress by default.

If a new module does not clearly support the Command Center, do not create it.
