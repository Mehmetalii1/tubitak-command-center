# CHANGE_CONTROL.md

## Change Control

Meaningful changes require:

1. a stated goal
2. allowed files
3. forbidden files
4. acceptance criteria
5. checks to run
6. `STATE.md` update when appropriate

## Meaningful Changes

- new folder or file structure
- changed phase boundary
- new approved decision
- new pending user decision
- completed setup task
- activated or deactivated gate
- added evidence source
- changed roadmap phase
